1. Ringkasan Aplikasi

  • Nama: Web App Kasir (Rice Bowl Monster / RBM)
  • Jenis: Aplikasi kasir (POS), input poin, klaim voucher, laporan, dan modul RBM Pro (absensi, petty cash, inventaris, pembukuan, dll.).
  • Entry point: login.html → setelah login redirect ke index.html.
  • Navigasi: Bukan SPA; setiap menu adalah link langsung ke file HTML. Dashboard (index.html) memakai iframe untuk konten (default: orders.html).
  • Data: Firebase Realtime Database (utama), localStorage (sesi & pengaturan), file JSON via server Node (opsional), Google Sheets via Google Apps Script.

4. Logika Bisnis Utama

4.1 Login

  1. User buka login.html, input username & password.
  2. Owner (hardcoded): burhan / 160502 → simpan rbm_user (role owner, outlet all); ensureDefaultFirebaseConnection(); redirect ke index.html.
  3. User lain: Baca rbm_users dari localStorage; cocokkan username (case-insensitive) dan password; simpan rbm_user; ensureDefaultFirebaseConnection(); redirect.
  4. app.js (DOMContentLoaded): jika tidak ada rbm_user dan bukan login.html, redirect ke login.html.

4.2 Kasir (POS)

  1. kasir.html: Init Firebase/adapter, muat categories dan products/{outletId}.
  2. Cart di memori; tambah/kurang item, ubah qty.
  3. Bayar: hitung subtotal, diskon, pajak, total; pilih metode bayar; generate order ID lewat counters/{outletId}; simpan ke orders/{outletId}/{orderId} status "Sudah Dibayar"; update pos_promos; cetak struk (print-bridge); kosongkan cart; modal kembalian jika tunai.
  4. Order ID: format {prefix}-{nomor}, mis. ORD-0001.

4.3 Menu

manage-menu.html: Kategori dari Firebase categories (cache rbm_menu_categories). Produk dari products/{outletId}. CRUD ke Firebase; listener realtime. menu.js: Hanya RBM_MENU_DATA statis (referensi).

4.4 Rewards / poin

  • Input poin: Cari customer via GAS ?phone=...; nominal → poin = nominal ÷ 100; APIService.addPoints() → POST GAS addPoints.
  • Klaim voucher: POST GAS validateClaimVoucher / validateClaimCode / generateClaimCode.
  • Hadiah: Firebase rewards; tambah = push; debet poin & kode klaim di backend GAS.

4.5 Laporan

revenue-report.html: Query orders/{outlet} filter date/status; hitung total, per metode bayar, chart; link ke report-transactions. report-transactions.html: List transaksi dari Firebase; filter tanggal/outlet/kasir; rbm_payment_methods dari localStorage.

4.6 Integrasi GAS

Code.gs: doGet + fungsi ke Sheet (Petty Cash, Database, Inventaris, Pembukuan, Pengajuan, Bank). google-apps-script-enhanced.js: doGet/doPost untuk users, poin, voucher, claim, menu (action: register, getUserData, addPoints, validateClaimCode, generateClaimCode, getMenuData, dll.).

6. Server & Backend

6.1 server.js (port 3001)

  • GET /db: Baca seluruh file database JSON.
  • POST /db: Terima body JSON; tulis ke file database.
  • GET /info: { status, port, dbFile }.
  • Path DB: server-config.json, atau argumen, atau database.json.

6.2 print-bridge.js (port 3000)

POST /print: Terima konten struk; kirim ke printer thermal via LAN (socket port 9100).

6.3 Google Apps Script

Code.gs: Satu Spreadsheet untuk Petty Cash, Database, Inventaris, Pembukuan, Pengajuan, Bank. Web App (SCRIPT_URL): Sheet terpisah users, poin, voucher, claim, menu.

7. Konfigurasi & Environment

  • app.js: CONFIG.SCRIPT_URL = URL deploy GAS Web App (input poin & claim voucher).
  • AUTH.USERS (app.js): User hardcoded modul input point (RBMPONTI, RBMDARMO); login → rbm_user.
  • login.html: Owner hardcoded burhan / 160502 (tidak di rbm_users).
  • server.js: CUSTOM_DB_PATH = null atau path penuh; server-config.json simpan path.
  • Firebase: Konfigurasi di custom-ui.js; ensureDefaultFirebaseConnection().

8. Cara Menjalankan & Build

  • Print Bridge: npm start → node print-bridge.js (port 3000).
  • Server database: node server.js (port 3001); optional: node server.js "D:\Path\database.json".
  • Front-end: Buka HTML langsung atau server statis; pastikan Firebase dan GAS URL benar.
  • Tidak ada build step; plain HTML/JS; npm install untuk server.
